# Transactions

All member earning and redemption activity is recorded as transactions in Fidivio. The Transactions module provides a complete, auditable record of every points movement.

## Transaction Types

| Type | Description |
| --- | --- |
| **Earn** | Points awarded to a member for a qualifying action |
| **Redeem** | Points deducted when a member claims a reward |
| **Adjustment - Credit** | Manual points added by an administrator |
| **Adjustment - Debit** | Manual points removed by an administrator |
| **Reversal** | Points reversed due to a return or error |
| **Transfer** | Points moved between accounts or programs |
| **Expiry** | Points removed due to expiration |
| **Bonus** | Points awarded from a campaign or promotion |

## Viewing Transactions

### All Transactions

Go to **Transactions \> All Transactions** to see a global list with filters:

- Date range
- Transaction type
- Program
- Status (Pending / Confirmed / Reversed)
- Member search
- Amount range

### Member-Level Transactions

Open a member profile and navigate to the **Transactions** tab for that member’s complete history.

## Transaction Details

Click any transaction to view full details:

| Field | Description |
| --- | --- |
| **Transaction ID** | Unique system identifier |
| **External Reference** | Your POS/e-commerce order ID |
| **Member** | Member who triggered the transaction |
| **Type** | Transaction type |
| **Points** | Points issued or deducted |
| **Transaction Value** | Monetary value of the original purchase |
| **Channel** | Where the transaction originated (POS, Web, App, API) |
| **Location** | Store or location code |
| **Status** | Current status of the transaction |
| **Timestamp** | Date and time of the transaction |
| **Applied Rules** | List of earning/redemption rules that applied |

## Processing a Manual Transaction

For transactions that didn’t capture automatically (e.g., missed receipt uploads):

1. Go to **Transactions \> All Transactions \> \+ New Transaction**
2. Select **Manual Transaction**
3. Search for and select the member
4. Enter transaction details:
   - Transaction date
   - Transaction value
   - Location/channel
   - External reference number
5. The system calculates points based on applicable earning rules
6. Review calculated points
7. Submit the transaction

<Note>
  Manual transactions may require supervisor approval depending on your configuration. Check **Settings \> Approvals** to configure approval workflows.
</Note>

## Reversing a Transaction

To reverse an earn transaction (e.g., customer return):

1. Find the original transaction
2. Click **Actions \> Reverse Transaction**
3. Enter the reversal reason
4. Specify partial or full reversal
5. Confirm

The system will:

- Deduct the reversed points from the member’s balance
- If the member doesn’t have sufficient balance, the balance will go to zero (cannot go negative by default)
- Record the reversal linked to the original transaction

## Pending Approvals

If your organization uses approval workflows, transactions requiring approval appear in **Transactions \> Pending Approvals**.

Approvers can:

- **Approve**: Process the transaction
- **Reject**: Decline with a reason
- **Request More Info**: Send back to submitter with questions

## Exporting Transactions

Export transaction data for external reporting:

1. Apply desired filters
2. Click **Export**
3. Select format: CSV or Excel
4. Choose columns to include
5. Download the file

<Tip>
  Schedule regular automated exports under **Settings \> Integrations \> Data Exports** to push transaction data to your data warehouse.
</Tip>
