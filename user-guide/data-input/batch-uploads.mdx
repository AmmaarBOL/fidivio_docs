---

Each data category has its own downloadable CSV template. All templates share a common structure.

## File requirements

- Save files in CSV format (`.csv`)
- First column is always `action` — insert, update, or delete
- Second column is always `id` — unique record identifier managed by Fidivio
- Remaining columns are category-specific

## Action types

| Action | Description |
| --- | --- |
| `insert` | Adds a new record. Leave `id` blank. |
| `update` | Overwrites an existing record. Provide the record `id`. |
| `delete` | Removes a record. Provide the record `id`. |

<Note>
The `id` value is auto-generated by Fidivio at insertion. To obtain `id` values for update or delete, export the current dataset from the relevant tab.
</Note>

## Common fields (all categories)

| Field | Definition | Format | Required |
| --- | --- | --- | --- |
| `action` | Operation: insert, update, or delete | `insert` \| `update` \| `delete` | Yes |
| `id` | Unique record identifier. Required for update and delete | Integer | Conditional |
| `transaction_date` | Date associated with the record | `yyyy-mm-dd` | Yes |
| `external_unique_identifier` | Optional cross-system reference ID | Integer | Optional |

See [Field definitions](/user-guide/reference/field-definitions) for category-specific fields.
