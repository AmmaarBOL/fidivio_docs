The Data Input section is where actual and budget data are uploaded into the model. Data can be entered manually or imported in bulk via CSV.

**Location:** Setup & Support → Data Input

## Seven data categories

<CardGroup cols={2}>
  <Card title="Accruals" href="/user-guide/data-categories/accruals">
    Points issued and yield per point.
  </Card>
  <Card title="Breakage Levels" href="/user-guide/data-categories/breakage-levels">
    Breakage rates, SSP, validity, and adjustments.
  </Card>
  <Card title="Expiry" href="/user-guide/data-categories/expiry">
    Points removed due to expiry events.
  </Card>
  <Card title="Other Revenue Sources" href="/user-guide/data-categories/other-revenue-sources">
    Non-points program revenue.
  </Card>
  <Card title="Program Costs" href="/user-guide/data-categories/program-costs">
    Operational program costs.
  </Card>
  <Card title="Redemptions" href="/user-guide/data-categories/redemptions">
    Points redeemed and cost per point.
  </Card>
  <Card title="Sales Information" href="/user-guide/data-categories/sales-information">
    Contract liability for conventional model programs.
  </Card>
</CardGroup>

## Input methods

<Tabs>
  <Tab title="Manual entry">
    Click **+ Add**, complete on-screen fields, and click **Add**.
  </Tab>
  <Tab title="CSV upload">
    Download the category template, populate it, and upload via the **Upload** option. See [CSV upload workflow](/user-guide/data-input/csv-upload-workflow).
  </Tab>
</Tabs>

## Recommended workflow

<Steps>
  <Step title="Prepare data">
    Gather source data and confirm partner codes are configured.
  </Step>
  <Step title="Download template">
    Download the CSV template for your data category.
  </Step>
  <Step title="Upload file">
    Upload via **Add → Upload**.
  </Step>
  <Step title="Validate">
    Check the **Batches** tab for processing status.
  </Step>
  <Step title="Fix errors">
    Download the handback file if errors occur and re-upload.
  </Step>
</Steps>
